## 项目概述

CatMusic 是一款基于 Android 平台开发的音乐播放器应用，旨在提供流畅、美观的音乐播放体验。本项目在原始开源版本的基础上，进行了本地化部署和二次开发，新增了本地 Node.js 代理服务与自定义歌曲功能。

## 一、核心功能与技术栈回顾

### 1. 功能特性

- 🎵 **音乐播放**：支持播放、暂停、上一首、下一首切换
- 🔄 **播放模式**：支持列表循环、单曲循环、随机播放
- 📊 **进度同步**：实时同步播放进度，支持拖动跳转
- 📝 **歌词同步**：支持歌词实时同步显示，自动滚动和高亮
- 🎨 **专辑封面**：流畅的封面旋转动画，精美的背景设计
- 🎼 **自定义歌曲**：支持通过本地 Node.js 服务器添加自定义/本地歌曲
- 📱 **响应式设计**：适配不同屏幕尺寸

### 2. 技术栈

- **Android + Java**：客户端开发
- **MediaPlayer / Service / Binder**：音乐播放与前后台通信
- **OkHttp / Glide**：网络请求与图片加载
- **Custom View（LyricView）**：自定义歌词显示控件
- **Node.js + Axios**：后端 QQ 音乐 API 代理 & 自定义歌曲服务

### 2. 环境与运行步骤（前后端联调）

1. **克隆仓库**
   ```bash
   git clone https://github.com/TF49/CatMusic.git
   cd CatMusic
   ```
2. **启动 Node.js 服务器**
   ```bash
   cd catmusic_server-main
   npm install
   node prod.server.js
   ```
   - 默认监听 `http://<局域网 IP>:3000/`
   - 确保 Android 设备与服务器处于同一局域网
3. **配置 Android 端服务器地址（按需）**
   - 默认由 `Config.getBaseUrl()` 在运行时解析：模拟器为 `10.0.2.2`，真机为本机私网 IPv4；真机访问局域网内其他电脑上的 Node 服务时，在 `res/values/strings.xml` 的 `catmusic_server_host` 中填写该电脑 IPv4。
4. **使用 Android Studio 运行 App**
   - 打开工程、等待 Gradle 同步
   - 连接真机/模拟器，点击运行

## 三、二次开发内容总结

### 1. 引入 Node.js 代理服务

- 在 `catmusic_server-main` 中实现了 QQ 音乐 API 的后端代理逻辑：
  - 负责请求签名、参数拼装和错误处理
  - 统一过滤付费歌曲、无时长歌曲等不适合播放的数据
  - 将 QQ 音乐的复杂返回结构整理为 Android 端可直接使用的简洁数据模型
- 通过 `prod.server.js` 启动本地服务，Android 端通过 `Config.getBaseUrl()` / `Config.getApiUrl(...)` 统一访问。

### 2. 支持自定义歌曲（本地资源接入）

- 在 `router.js` 中新增 `customSongs` 配置区域：
  - 使用自定义 `id`、`mid`、`name`、`singer`、`url`、`duration`、`pic`、`album` 等字段描述歌曲
  - `url`、`pic` 可以指向本地静态资源目录 `public/music`、`public/images`
- 在歌单接口、歌曲 URL 接口和歌词接口中，增加了对自定义歌曲的专项处理：
  - 歌单结果中将 `customSongs` 合并进返回列表
  - 请求播放 URL 时优先命中自定义歌曲的预设地址
  - 可为自定义歌曲配置预置歌词文本
- 编写了《添加自定义歌曲完整指南.md》，形成一套可复用的配置流程，便于后续继续扩展本地曲库。

### 3. Android 客户端联调与体验优化

- 抽象 `Config` 配置类，统一管理后端基础地址与接口路径；基础地址由运行时解析或 `strings.xml` 中的 `catmusic_server_host` 覆盖。
- 在 `HomeActivity`、`SongListActivity`、`PlayerActivity` 等页面中：
  - 完成与本地 Node.js 服务的接口联调与数据展示
  - 确保歌单 → 歌曲列表 → 播放器的跳转与数据传递链路顺畅
- 对歌词业务（如 `LyricBiz` 与 `LyricParser`）进行适配和优化：
  - 兼容后端返回的新数据格式
  - 保证自定义歌曲在歌词同步、高亮、滚动上的体验与线上歌曲一致。
- 适度优化播放器 UI：
  - 调整 `album_art_background.xml`、`activity_player.xml` 等资源
  - 提升封面背景、进度条与歌词区域的整体观感。

## 四、遇到的问题与解决思路

1. **QQ 音乐接口签名与跨域问题**
   - 通过后端 Node.js 代理统一处理请求与签名，前端只关心业务数据，避免直接调用第三方 API 带来的复杂性。
2. **自定义歌曲与线上数据结构对齐**
   - 在设计 `customSongs` 时完全复用 `handleSongList` 的字段结构，确保客户端无需区分“来自 QQ 音乐”还是“本地自定义”。
3. **局域网访问与设备兼容**
   - Android 端通过 `LanHostResolver` 与可选 `catmusic_server_host` 避免在代码中写死 IP；Node 端 `router.js` 使用 `os.networkInterfaces()` 生成 `SERVER_BASE`，自定义资源的 `pic`/`url` 与当前机器局域网 IP 一致。
4. **歌词解析兼容性**
   - 在歌词解析逻辑中，增加对空歌词、格式不规范歌词的保护性处理，避免导致界面异常或崩溃。
