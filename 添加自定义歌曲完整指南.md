# 在CatMusic项目中添加自定义歌曲的完整指南

## 方案概述

本指南将详细介绍如何在CatMusic项目中添加自定义歌曲，使您能够扩展系统中的歌曲库，不受限于API返回的有限歌曲数量。

## 系统架构回顾

CatMusic使用以下架构：
1. Android客户端 → 请求Node.js服务器（位于 `D:\Developer Tool\AndroidStudioProjects\CatMusic\catmusic_server-main`）
2. Node.js服务器 → 代理请求到QQ音乐API
3. QQ音乐API → 返回原始歌曲数据
4. Node.js服务器 → 处理数据并添加自定义歌曲 → 返回给Android客户端

### 第四步：处理歌曲URL获取接口

为了让自定义歌曲也能正确获取播放URL，需要在 `registerSongsUrl` 函数中添加处理逻辑。

在 `registerSongsUrl` 函数中（大约在第356-432行），在函数开头添加自定义歌曲URL处理：

在这一行：
```javascript
const mid = req.query.mid
```

后面添加：
```javascript
// 检查是否包含自定义歌曲的MID
const customMids = customSongs.map(song => song.mid);
const hasCustomMids = Array.isArray(mid) ? mid.some(m => customMids.includes(m)) : customMids.includes(mid);

if (hasCustomMids) {
  // 处理自定义歌曲的URL
  const urlMap = {};
  
  // 为自定义歌曲设置预设的URL
  customSongs.forEach(song => {
    if ((Array.isArray(mid) && mid.includes(song.mid)) || mid === song.mid) {
      urlMap[song.mid] = song.url || ''; // 使用预设的URL，如果没有则为空
    }
  });

  // 处理非自定义歌曲的请求
  let nonCustomMids = [];
  if (Array.isArray(mid)) {
    nonCustomMids = mid.filter(m => !customMids.includes(m));
  } else {
    nonCustomMids = customMids.includes(mid) ? [] : [mid];
  }
  
  if (nonCustomMids.length > 0) {
    // 为非自定义歌曲调用原来的处理逻辑
    let midGroup = [];
    if (mid.length > 100) {
      const groupLen = Math.ceil(mid.length / 100)
      for (let i = 0; i < groupLen; i++) {
        midGroup.push(mid.slice(i * 100, (100 * (i + 1))))
      }
    } else {
      midGroup = [mid]
    }

    // 处理返回的 mid
    function process(mid) {
      const data = {
        req_0: {
          module: 'vkey.GetVkeyServer',
          method: 'CgiGetVkey',
          param: {
            guid: getUid(),
            songmid: mid,
            songtype: new Array(mid.length).fill(0),
            uin: '0',
            loginflag: 0,
            platform: '23',
            h5to: 'speed'
          }
        },
        comm: {
          g_tk: token,
          uin: '0',
          format: 'json',
          platform: 'h5'
        }
      }

      const sign = getSecuritySign(JSON.stringify(data))
      const url = `https://u.y.qq.com/cgi-bin/musics.fcg?_=${getRandomVal()}&sign=${sign}`

      return post(url, data).then((response) => {
        const data = response.data
        if (data.code === ERR_OK) {
          const midInfo = data.req_0.data.midurlinfo
          const sip = data.req_0.data.sip
          const domain = sip[sip.length - 1]
          midInfo.forEach((info) => {
            // 获取歌曲的真实播放 URL
            urlMap[info.songmid] = domain + info.purl
          })
        }
      })
    }

    // 构造多个 Promise 请求
    const requests = midGroup.map((mid) => {
      return process(mid)
    })

    // 并行发送多个请求
    return Promise.all(requests).then(() => {
      // 所有请求响应完毕，urlMap 也就构造完毕了
      res.json({
        code: ERR_OK,
        result: {
          map: urlMap
        }
      })
    })
  } else {
    // 只有自定义歌曲
    res.json({
      code: ERR_OK,
      result: {
        map: urlMap
      }
    });
  }
}
```

### 第六步：配置自定义歌曲参数

现在您可以通过修改 `customSongs` 数组来配置自定义歌曲：

#### 配置参数详解：

1. **id** (数字): 歌曲的唯一标识符，建议使用大数字避免冲突（如999999001开始）
2. **mid** (字符串): 歌曲的MID，必须唯一，用于识别歌曲（如"my_custom_song_001"）
3. **name** (字符串): 歌曲名称
4. **singer** (字符串): 歌手名称
5. **url** (字符串): 歌曲播放URL，必须是可以直接播放的音频文件链接
6. **duration** (数字): 歌曲时长，单位为秒
7. **pic** (字符串): 专辑封面图片URL
8. **album** (字符串): 专辑名称

#### 示例配置：

```javascript
const customSongs = [
  {
    id: 999999001,
    mid: "my_custom_song_001",
    name: "夜空中最亮的星",
    singer: "逃跑计划",
    url: "http://your-music-server.com/songs/night_star.mp3",  // 您的音乐文件URL
    duration: 270,  // 4分30秒
    pic: "https://y.gtimg.cn/music/photo_new/T002R800x800M000001ngxPw4Polma.jpg?max_age=2592000",  // 专辑封面URL
    album: "世界"

  }
];
```

### 第七步：准备音乐文件和资源

1. **音乐文件**: 确保您有可公开访问的音乐文件URL
2. **专辑封面**: 准备专辑封面图片，建议使用高质量图片
3. **歌词文件**: 如果需要歌词，准备LRC格式的歌词

### 第八步：启动服务器并测试

1. 保存所有修改
2. 重启Node.js服务器：
```bash
cd "D:\Developer Tool\AndroidStudioProjects\CatMusic\catmusic_server-main"
node prod.server.js
```
3. 在Android应用中测试自定义歌曲是否正常显示和播放

### 第九步：故障排除

如果自定义歌曲没有出现：

1. **检查服务器日志**: 确认服务器正常启动且没有错误
2. **检查MID唯一性**: 确保自定义歌曲的MID与现有歌曲不冲突
3. **检查URL有效性**: 确认音乐文件URL可以正常访问
4. **检查网络连接**: 确认Android应用能正常连接服务器

## 注意事项

1. **版权问题**: 请确保您有权使用自定义歌曲的音频文件
2. **文件格式**: 音乐文件建议使用MP3或AAC格式以确保兼容性
3. **服务器容量**: 考虑到存储和带宽限制，合理管理自定义音乐文件
4. **定期维护**: 定期检查自定义歌曲的URL有效性

通过以上步骤，您就可以成功地在CatMusic系统中添加自定义歌曲，扩展您的音乐库，摆脱API返回歌曲数量的限制。
