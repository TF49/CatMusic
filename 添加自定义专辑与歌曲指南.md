# 添加自定义专辑与歌曲的完整指南

## 功能概述

在原有「自定义歌曲出现在所有专辑中」的基础上，本方案实现**自定义专辑 → 自定义歌曲**的层级结构：

1. **自定义专辑**：在首页推荐列表中展示您创建的专辑
2. **自定义歌曲**：歌曲归属于特定自定义专辑，仅在进入该专辑时显示

这样可以将自定义内容按专辑分类组织，避免自定义歌曲混入所有歌单。

---

## 实现思路

### 数据流对比

| 对比项 | 原方案 | 新方案 |
|--------|--------|--------|
| 自定义歌曲出现位置 | 所有专辑的歌单中 | 仅对应的自定义专辑中 |
| 专辑来源 | 仅来自 QQ 音乐 API | QQ 音乐 + 自定义专辑 |
| 专辑-歌曲关系 | 无归属关系 | 一对多（专辑→歌曲） |

### 架构说明

```
首页推荐 (getRecommend)
    ├── QQ 音乐专辑 1, 2, 3...
    └── 自定义专辑 A, B, C...  ← 新增

点击专辑 → 歌曲列表 (getAlbum?id=xxx)
    ├── 若 id 为 QQ 专辑 → 调用 QQ API + 不再追加全局自定义歌曲
    └── 若 id 为自定义专辑 → 直接返回该专辑下的自定义歌曲
```

---


### 第二步：修改 getRecommend 接口

在 `registerRecommend` 中，将自定义专辑追加到推荐列表：

找到返回数据的代码（约 228-234 行）：

```javascript
res.json({
  code: ERR_OK,
  result: {
    sliders,
    albums
  }
})
```

修改为：

```javascript
// 将自定义专辑追加到推荐列表前面（也可改为 albums.concat(customAlbumsForList) 放后面）
const customAlbumsForList = customAlbums.map(a => ({
  id: a.id,
  title: a.title,
  username: a.username,
  pic: a.pic
}));
const allAlbums = [...customAlbumsForList, ...albums];

res.json({
  code: ERR_OK,
  result: {
    sliders,
    albums: allAlbums
  }
})
```

### 第三步：修改 getAlbum 接口

在 `registerAlbum` 中，区分「自定义专辑」和「QQ 专辑」：

将原来的逻辑：

```javascript
post(url, data).then((response) => {
  const data = response.data
  if (data.code === ERR_OK) {
    const list = data.req_0.data.songlist
    const songList = handleSongList(list)
    const allSongs = songList.concat(customSongs)  // 所有专辑都追加自定义歌曲
    res.json({
      code: ERR_OK,
      result: { songs: allSongs }
    })
  } else {
    res.json(data)
  }
})
```

修改为：

```javascript
const albumId = Number(req.query.id);

// 判断是否为自定义专辑
if (customAlbumIds.has(albumId)) {
  const album = customAlbums.find(a => a.id === albumId);
  return res.json({
    code: ERR_OK,
    result: {
      songs: album ? album.songs : []
    }
  });
}

// 否则按原逻辑请求 QQ 音乐（且不再追加自定义歌曲）
post(url, data).then((response) => {
  const data = response.data
  if (data.code === ERR_OK) {
    const list = data.req_0.data.songlist
    const songList = handleSongList(list)
    res.json({
      code: ERR_OK,
      result: {
        songs: songList   // 只返回 QQ 歌曲，不再追加 customSongs
      }
    })
  } else {
    res.json(data)
  }
})
```

### 第四步：修改 registerSongsUrl

将原先使用的 `customSongs` 全部替换为 `allCustomSongs`，其余逻辑不变。

### 第五步：修改 registerLyric

将原先使用的 `customSongs` 全部替换为 `allCustomSongs`，其余逻辑不变。

## 数据结构说明

### 自定义专辑字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 专辑唯一 ID，建议 999000001 起，避免与 QQ 冲突 |
| title | string | 专辑标题（展示在首页） |
| username | string | 创建者/歌单主，可填「我」等 |
| pic | string | 专辑封面 URL |
| songs | array | 该专辑下的自定义歌曲列表 |

### 自定义歌曲字段（与原有一致）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 歌曲唯一 ID |
| mid | string | 歌曲 MID，必须唯一 |
| name | string | 歌曲名 |
| singer | string | 歌手 |
| url | string | 播放地址 |
| duration | number | 时长（秒） |
| pic | string | 封面图 URL |
| album | string | 专辑名称（建议与自定义专辑 title 一致） |

---

## 迁移说明

若您已经在使用旧的 `customSongs` 配置：

1. 创建一个自定义专辑（如「我的收藏」）
2. 将原有 `customSongs` 中的歌曲移入该专辑的 `songs` 数组
3. 删除原来的 `customSongs` 常量
4. 按上述步骤修改 `registerRecommend`、`registerAlbum`、`registerSongsUrl`、`registerLyric`

---

## 注意事项

1. **ID 规划**：自定义专辑 id 建议 999000001 起，歌曲 id 建议 999999001 起
2. **mid 唯一性**：所有自定义歌曲的 `mid` 在全局必须唯一
3. **URL 可访问性**：`url`、`pic` 需确保 Android 客户端能访问（内网需同一网段）
4. **Android 端无需改动**：首页和歌单页都通过 id 请求，只要服务端返回格式一致即可
